- ISSUE Automation (หลังบ้าน)
1. วางแผนการทำ Script Test (ต้องให้สคริปทำงานได้ตรงตามโฟลของระบบ เน้น Best Case) ละเอียด และสามารถใช้งาน Script ได้ทุกครั้งเมื่อมีการ Run ทดสอบ
2. การ Dev Script ต้อง อ้างอิงจาก ID เป็นหลักเสมอ เพื่อป้องกัน Value เปลี่ยน เช่น x-path / Locator ไม่ควรใช้ / dropdown value ต้องอ้างอิง ID ของแต่ละชั้น
3. เรื่องของการ Comment Script ต้องระบุเหตุการณ์ให้ละเอียดว่า บรรทัดนี้ ทำอะไร ดึงข้อมูลอะไร เก็บไว้ที่ตัวแปลอะไร นำไปใช้ทำอะไรต่อ
4. ระยะเวลาในการทำ Script ที่เป็นระดับ Integration Testing นั้นค่อนข้างใช้เวลานาน และต้องมีสมาธิในการคิดและจำ Step Script Testing ต้องครอบคลุมในทุก Function ในเมนูนั้นๆ เพื่อเก็บไว้เป็นกลุ่มเมนูในการนำไปทำ System Test ที่ทำงานคู่กับระบบขายหน้าบ้าน
5. เรื่องของการ Review script อาจจะต้องมีคนแนะนำได้ เช่น คำสั่งในการใช้งาน ดับ Normal - Advance  หรือ  Function ต่างๆของ Playwright ที่ทำได้ หรือไกด์อื่นๆเป็นข้อมูล เนื่องจากบางทีใช้ Chat GPT ได้คำตอบหรือผล ที่ตรงไม่ 100% แต่ก็ช่วยเป็นไกด์ได้ระดับนึงเพือมาคิดต่อว่าจะสามารถไปทางไหนได้
6. กลับไปศึกษา Automation Deploy (ข้อนี้ต้องมั่นใจว่า Script Test Automation ผ่านการทำงานได้ครบ Loop การทดสอบแล้วเท่านั้น) 
7. การแบ่ง Folder วาง Script File ใน GitHub ที่เป็น Repository ของ Tester โดยเฉพาะ  จะ Commit กรณีที่มีการ Run Local แล้วผ่านทุกเคสแล้วเท่านั้น
